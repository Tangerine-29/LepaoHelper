# Contributing Guide

Thanks for your interest in contributing to LepaoHelper.

## Before You Start

- Open an issue first for large changes.
- Keep pull requests focused and minimal.
- Follow existing coding style and naming.

## Development Setup

1. Install Theos on macOS.
2. Clone this repository.
3. Run `make package` to verify local build.

## Commit Style

Use clear, action-oriented commit messages, for example:

- `fix: avoid nil crash in account import`
- `feat: add runtime toggle for diagnostics`

## Pull Request Checklist

- [ ] Build passes locally (`make package`)
- [ ] No unrelated files changed
- [ ] Documentation updated when needed
- [ ] Behavior/risk changes clearly described

## Code of Conduct

By participating, you agree to follow `CODE_OF_CONDUCT.md`.
